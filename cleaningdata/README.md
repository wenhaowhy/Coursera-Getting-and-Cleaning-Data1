<h2><a id="user-content-purpose-" class="anchor" href="#purpose-" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>Purpose </h2>

<p>The purpose of this project is to demonstrate your ability to collect, work with, and clean a data set. The goal is to prepare tidy data that can be used for later analysis. You will be graded by your peers on a series of yes/no questions related to the project. You will be required to submit: </p>

<ol>
<li>a tidy data set as described below;</li>
<li>a link to a Github repository with your script for performing the analysis; and </li>
<li>a code book that describes the variables, the data, and any transformations or work that you performed to clean up the data called <code>CodeBook.md</code>. </li>
</ol>

<p>You should also include a <code>README.md</code> in the repo with your scripts. This repo explains how all of the scripts work and how they are connected.  </p>

<h2><a id="user-content-objectives" class="anchor" href="#objectives" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>Objectives</h2>

<p><code>run_analysis.R</code> performs the following:</p>

<ol>
<li>Merges the training and the test sets to create one data set.</li>
<li>Extracts only the measurements on the mean and standard deviation for each measurement. </li>
<li>Uses descriptive activity names to name the activities in the data set</li>
<li>Appropriately labels the data set with descriptive activity names. </li>
<li>Creates a second, independent tidy data set with the average of each variable for each activity and each subject. </li>
</ol>

<h2><a id="user-content-run_analysisr" class="anchor" href="#run_analysisr" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1h-1c-1.5 0-3-1.69-3-3.5s1.55-3.5 3-3.5h4c1.45 0 3 1.69 3 3.5 0 1.41-0.91 2.72-2 3.25v-1.16c0.58-0.45 1-1.27 1-2.09 0-1.28-1.02-2.5-2-2.5H4c-0.98 0-2 1.22-2 2.5s1 2.5 2 2.5z m9-3h-1v1h1c1 0 2 1.22 2 2.5s-1.02 2.5-2 2.5H9c-0.98 0-2-1.22-2-2.5 0-0.83 0.42-1.64 1-2.09v-1.16c-1.09 0.53-2 1.84-2 3.25 0 1.81 1.55 3.5 3 3.5h4c1.45 0 3-1.69 3-3.5s-1.5-3.5-3-3.5z"></path></svg></a>run_analysis.R</h2>

<ol>
<li>It downloads the <strong>UCI HAR Dataset</strong> data set and puts the zip file working directrory. After it is downloaded, it unzips the file into the UCI HAR Dataset folder. </li>
<li>It loads the <strong>train</strong> and <strong>test</strong> data sets and appends the two datasets into one data frame. This is done using <code>rbind</code>.</li>
<li>It extracts just the <em>mean</em> and <em>standard deviation</em> from the <strong>features</strong> data set. This is done using <code>grep</code>.</li>
<li>After cleaning the column names, these are applied to the <strong>x</strong> data frame.<br></li>
<li>After loading <strong>activities</strong> data set, it converts it to lower case using <code>tolower</code> and removes underscore using <code>gsub</code>. <em>activity</em> and <em>subject</em> column names are named for <strong>y</strong> and <strong>subj</strong> data sets, respectively.</li>
<li>The three data sets, <strong>x</strong>, <strong>y</strong> and <strong>subj</strong>, are merged. Then, it is exported as a <em>txt</em> file into the Project folder in the same working directory, named <strong>merged.txt</strong>.</li>
<li>The <em>mean</em> of activities and subjects are created into a separate tidy data set which is exported into the Project folder as <em>txt</em> file; this is named <strong>average.txt</strong>.</li>
</ol>

<p><em>The R code contains <code>str</code> for easier preview of the two final data sets.</em></p>
